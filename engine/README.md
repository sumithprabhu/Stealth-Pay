# engine

> **V1 reference — not used in production.**
>
> The active architecture (V2) uses client-side ZK proofs generated by the SDK. This folder contains the original TEE-based execution service kept for historical reference.

## What it was

An off-chain service that ran inside Intel TDX enclaves on 0G Compute nodes. It handled encrypted intent routing, private state transitions, and generated on-chain attestations on behalf of users.

V2 removed the need for a trusted server entirely — users now hold their own spending keys and generate ZK proofs locally.

## Structure

```
engine/src/
├── core/
│   ├── CryptoEngine.ts         — encrypted note state transitions
│   ├── AttestationSigner.ts    — enclave ECDSA attestation signing
│   └── NoteManager.ts          — note read/write via 0G Storage
├── chain/
│   └── PrivacyPoolClient.ts    — V1 contract interaction
├── storage/
│   └── ZeroGStorage.ts         — 0G Storage adapter
└── api/
    └── server.ts               — HTTP API for SDK clients
```
